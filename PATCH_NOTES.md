# Undiscord Python GUI 패치 내역 (Patch Notes)

본 문서는 파이썬 기반 Undiscord GUI 도구의 최신 패치 및 리팩토링, 보안 최적화 내역을 다른 팀원들과 공유하고 공지하기 위해 기록되었습니다.

---

## 📌 패치 개요
- **최신 버전**: V1.5.1
- **릴리즈 날짜**: 2026-07-12
- **주요 목표**: 삭제된 메시지 PC 백업 지원, 화면 노출 방지(토큰 및 로그 마스킹) 및 Ed25519 서명 기반 동적 SSL 피닝 갱신 도입

## 🛠️ 주요 변경 및 개선 사항 (V1.5.0)

### 1. 삭제 예정 메시지 로컬 PC 자동 백업 기능
- 디스코드에서 영구히 제거되기 직전, 삭제 대상 메시지(시간, 작성자, 내용) 정보를 로컬 저장소인 `deleted_backups/` 디렉터리에 보기 좋은 채팅 로그 복사본 형태로 안전하게 백업 및 보관합니다. 메시지 ID는 제외하고 `[시간][글쓴사람][내용]` 형식으로 구분선이나 무의미한 줄바꿈 없이 연속되게 기록하여, 실제 채팅 내용을 복사해 붙여넣은 것처럼 편리하게 대화 로그를 조회할 수 있습니다.

### 2. 토큰 입력 은닉 및 화면 노출 차단 (Security Shield)
- GUI 텍스트 박스에 로그인 토큰(열쇠)을 입력 혹은 가져오는 순간, 화면상에는 고정된 마스킹 문자열(`••••••••••••••••`)로만 치환하여 노출 위험을 원천 차단했습니다.
- 비밀번호 표시 토글(보기/숨기기) 버튼을 제공해 필요할 때만 일시적으로 실 토큰을 노출하며, 포커스를 벗어나면 자동으로 재마스킹됩니다.

### 3. 실시간 진행 로그 내 채팅 본문 마스킹 기능
- 대시보드의 터미널 형태 로그창과 삭제 시작 전 표시되는 삭제 대상 요약 모달창에서 내 채팅 내용을 `●●● (마스킹됨)` 형태로 숨겨주는 기능을 구현했습니다.
- 화면 공유, 인터넷 라이브 방송, 스크린샷 캡처 시 개인 대화 정보가 무단 노출되는 현상을 완벽히 차단합니다. 로컬 PC에 백업되는 파일에는 원본 텍스트가 정상 기록되므로 안심하셔도 됩니다.

### 4. Ed25519 전자 서명 검증 기반 동적 SSL 피닝(Pinning) 갱신
- 중간자 공격(MITM) 및 사설 인증서 도청 위협에 대비하는 SSL 피닝을 고도화했습니다.
- 애플리케이션 시작 시 원격 보안 서버에서 최신 SSL 인증서 지문 목록(`cert_pins.json`)을 안전하게 동적 갱신 및 메모리에 주입합니다.
- 변조된 데이터 유입을 차단하기 위해 **Ed25519 비대칭키 전자 서명 검증**을 수행하며, 서명이 위조되었거나 네트워크 에러 시 안전하게 기본 내장 핀으로 Fallback 처리됩니다.

### 5. 간편 로그인(Easy Login) IPC 통신 보안 정책 강화
- 서브프로세스와 부모 GUI가 토큰을 전달하는 내부 채널에서 비밀키(`ENV_SEC_KEY`) 유효성을 검사하여 평문 상태의 원시 토큰이 전송되는 현상을 사전에 방어하고, 누출 시 전송을 전면 차단합니다.

## 🛠️ 이전 변경 및 개선 사항 (V1.4.0)

### 1. 간편 로그인 수동 종료 시 예외 충돌 해결
- 디스코드 간편 로그인 화면이 기동된 상태에서, 사용자가 로그인을 완료하지 않고 웹뷰 창을 직접 닫을 때 발생하는 `System.ObjectDisposedException` (WebView2 객체 소멸) 예외가 메인 GUI 대시보드 로그에 시각적 에러로 쏟아지는 현상을 방어했습니다.
- **감시 루프 즉시 이탈**: `undiscord_login.py` 내의 예외 감지 키워드 목록에 `disposed`, `webview`, `webview2`를 추가하여 창이 소멸하는 시점에 감시 루프가 자원을 즉시 해제하고 조기 종료되도록 조치했습니다.
- **마이너 에러 로그 필터링**: 부모 프로세스인 `undiscord_gui.py`가 서브프로세스로부터 오류 로그를 수집할 때, 사용자가 창을 닫아 발생한 단순 예외나 무해한 User-Agent 경고 문구 등은 붉은색 에러 로그 대신 일반 취소(warn) 안내로 깔끔하게 UI에 노출하고, 세부 로그는 백그라운드 콘솔로만 보존하도록 정돈했습니다.

### 2. 크롬 미설치 대비 다중 브라우저 로그인 폴백(대체 보안책) 구현
- 시스템에 WebView2 런타임이 없거나 간편 로그인 뷰어 창을 구동할 수 없는 환경에 대비해, 대체 수단으로 다중 독립 실행형 브라우저를 직접 띄울 수 있는 창을 구현했습니다.
- **[undiscord_browser.py] 신설**: Windows 레지스트리(`App Paths`) 및 기본 프로그램 경로를 감지하여 Google Chrome, Microsoft Edge, Mozilla Firefox 브라우저의 설치 여부와 실행 파일 경로(`*.exe`)를 추출하고 비동기적으로 실행시키는 기능을 구현했습니다.
- **로그인 브라우저 선택 다이얼로그 탑재**: 간편 로그인 뷰어 기동에 실패할 시, 다크 테마 디자인 톤에 맞춘 세련된 브라우저 선택 모달 창이 팝업됩니다. 
- **설치 기반 버튼 활성화**: 감지된 브라우저 버튼만 선명하게 활성화(Enabled)하고 미설치된 브라우저는 비활성화(Disabled)하여 직관성을 높였습니다.
- **수동 토큰 가이드 연동**: 브라우저 실행 버튼을 누르면 해당 브라우저로 디스코드 로그인 페이지가 뜸과 동시에, 메인 창의 토큰 획득 도움말(F12 가이드) 팝업이 함께 연계 호출되어 매끄럽게 토큰을 복사 및 입력할 수 있도록 동선을 설계했습니다.

### 3. 만든이(LibraLight) 라벨 연결 주소 수정
- 프로그램 하단 정보 패널의 만든이 `LibraLight` 텍스트 라벨 클릭 시 열리던 디스코드 초대 링크를 본 프로그램의 공식 GitHub 저장소 주소(`https://github.com/LibraLightLuda/ClearDiscord`)로 연결되도록 변경하여 사용자와의 피드백 채널을 일원화했습니다.

## 🛠️ 이전 변경 및 개선 사항 (V1.2.0)

### 1. 소스 코드 모듈화 및 리팩토링 (가독성 및 토큰 절약)
기존 단일 1,500줄 이상의 비대했던 `undiscord_gui.py` 파일을 역할별 논리 단위로 분리하여 코드의 유지보수 효율을 높이고 AI 분석에 효율적인 가벼운 모듈 형태로 분할하였습니다.
- **[undiscord_utils.py](file:///d:/Antigravity/undiscord/undiscord/undiscord_utils.py)**: 의존 라이브러리 자동 설치, Snowflake 변환, 시간 포맷팅, 토큰 보안용 AES 암복호화 및 URL 화이트리스트 검사 전담.
- **[undiscord_core.py](file:///d:/Antigravity/undiscord/undiscord/undiscord_core.py)**: API 호출, Rate Limit 대기 및 메시지 검색/삭제 루프를 관리하는 핵심 엔진 `UndiscordCore` 분리.
- **[undiscord_dialogs.py](file:///d:/Antigravity/undiscord/undiscord/undiscord_dialogs.py)**: 마스터 비밀번호 설정 및 입력을 요구하는 모달 창 `PasswordDialog` 분리.
- **[undiscord_gui.py](file:///d:/Antigravity/undiscord/undiscord/undiscord_gui.py)**: 메인 레이아웃 및 제어 흐름만 남기고 상단 re-export 지정을 통해 기존 단위 테스트 및 모듈들과 100% 호환되도록 구성.

### 2. 비밀번호 선 인증 보안 체계 도입
로컬에 암호화 보존된 토큰 및 기타 설정값이 노출되는 것을 방지하기 위해 프로그램 기동 시 즉각 보호 프로세스를 수행합니다.
- **메인 창 초기 은닉**: 마스터 비밀번호 인증을 완전히 완료하거나 최초 설정을 마칠 때까지 메인 윈도우 창을 화면에 완전히 드러내지 않고 숨김(`withdraw`) 상태로 유지합니다.
- **강제 포커스 및 최상단 노출**: 모달 다이얼로그에 `topmost` 속성과 `focus_force`를 주어 실행 즉시 입력 창이 가장 위로 뜨도록 제어했습니다.
- **인증 성공 시 노출 / 실패 시 완전 차단**: 비밀번호 인증 성공 시에만 메인 창이 복구(`deiconify`)되며, 사용자가 입력을 취소하거나 3회 오류로 영구 파기되는 경우 즉시 프로세스를 파괴(`destroy`)하고 앱을 완전히 강제 종료합니다.

### 3. 고성능 네트워크 패킷 보안 적용 (오버헤드 0%)
통신 속도 저하 없이 패킷 가로채기나 페이크(피싱) 서버 도메인 접근을 차단하는 보안 수칙을 추가했습니다.
- **위조 도메인 방지 (URL 화이트리스트)**: 모든 API 통신 직전 `validate_discord_url` 함수를 강제 통과하여, 목적지 주소가 반드시 공식 디스코드 HTTPS API 주소(`https://discord.com/api/v9/`)로 시작되는지 엄격 검사합니다.
- **만인간 공격(MitM) 프록시 무력화**: `requests` 통신 세션 시 로컬 환경 프록시를 완전히 바이패스하도록 `proxies = {'http': None, 'https': None}` 및 `trust_env = False`를 적용해 패킷 분석/수정 도구의 하이재킹을 방어합니다.
- **명시적 SSL/TLS 유효성 검증 고정**: 모든 요청에 `verify=True` 검증을 강제 고정하여 암호화되지 않은 비보안 통신 유도 공격을 차단합니다.

### 4. UI/UX 레이아웃 최적화
- **창 세로 높이 조정**: 모니터의 DPI 스케일링 배율에 의해 하단 작업 제어 버튼(시작/중단)이 잘려서 보이지 않는 문제를 수정하기 위해 창 세로 크기를 확장 조정했습니다. (기본 `1120x860` 및 최소 `1080x800` 지정)
- **정보 통합형 헤더 디자인**: 최하단 푸터 영역을 완전히 제거하고, 좌측 하단의 로고 설명 란에 만든이 `LibraLight`(디스코드 채널로 연동) 및 오픈소스 원본 저장소 `victornpb/undiscord` 링크를 자연스럽게 결합하여 콤팩트한 구조를 갖추었습니다.

- **PyInstaller 빌드 탑재**: `build.bat` 스크립트를 변경하여 컴파일된 `UndiscordGUI.exe` 실행파일 자체 아이콘에도 차가운 색상 아이콘(`cold.ico`)이 각인되어 빌드되도록 최종 반영했습니다.

### 7. 마스터 비밀번호 설정 우회 기능 추가 ("사용안함" 옵션)
- 최초 실행 및 인증 토큰 입력 시 팝업되는 마스터 비밀번호 설정 창에 **[사용안함]** 버튼을 새롭게 디자인하여 배치했습니다.
- 이를 클릭하면 마스터 비밀번호 설정을 생략하고 토큰을 안전 모드 대신 평문 상태로 즉시 보존하여, 비밀번호 관리를 원치 않는 사용자가 원활하게 앱을 기동하도록 편의성을 높였습니다.

### 8. 서버 및 채널 자동 완성 조회 기능 추가 (귀찮은 복사/붙여넣기 제거)
- 디스코드 토큰 입력 시 사용자가 속한 서버(Guild) 목록과 텍스트 채널(또는 개인 DM) 목록을 실시간으로 가져와 선택할 수 있는 자동 완성 콤보박스를 `Card 2` 영역에 새로이 설계 및 추가했습니다.
- 백그라운드 스레드(threading.Thread)를 도입하여 API 조회 처리 시 UI 프레임이 굳는 멈춤 현상(UI Freeze)을 완전히 차단하고, 불러오는 중 상태("Loading...")와 완료 상태를 직관적으로 로그 및 인터페이스에 연동했습니다.

### 9. DM 채널 별명(global_name) 파싱 개선
- 개인 DM 및 그룹 DM을 불러올 때, 디스코드 사용자 계정에 수동으로 지정된 별명(global_name)이 있을 경우 이를 우선 파싱하여 `별명 (사용자명)` 형태로 리스트에 친절하게 렌더링되도록 시각적 기능을 보강했습니다.
